### seg_stats 출력 지표
`seg_stats(sub, min_n=5)` 함수가 반환하는 딕셔너리 키:
- `n`, `weighted` (가중치 합산)
- `profit_rate` — 영업이익률 중앙값 (%)
- `cost` — 원가율·인건비율·임차료율·기타비용율·영업이익률 각 중앙값 (%)
- `rent_rate` — 임차료율 중앙값 (% = `cost['임차료율']` 단축키, 버블차트에서 직접 참조)
- `hardship` — 애로사항 9개 항목 `{labels, rates, counts}`
- `closure_pct` — 폐업의향 비율 (운영계획코드 3·4 합산, %)
- `closure_dist` — 운영계획 4개 항목 분포 (%)
- `revenue_median` — 연간 매출 중앙값 (만원)
- `monthly_revenue` — 월 매출 중앙값 (만원 = revenue_median / 12)
- `monthly_profit` — 월 영업이익 중앙값 (만원)
- `neg_pct` — 영업이익 < 0 비율 (%)

`min_n` 파라미터: n < min_n 이면 수치 지표를 모두 None으로 반환 (n은 보존).
- 기본값 5 (일반 집계)
- SEC4 프랜차이즈×고용형태 셀: `min_n=15`
- SEC6 운영연수×고용형태 셀: `min_n=10`
- SEC7 시도별 집계: `min_n=20`

### 애로사항 코드 (경영_애로사항1~10코드)
**컬럼 구조**: 각 컬럼이 항목별 1/NaN 플래그 (1=해당, NaN=미해당) — 코드값이 아님.
집계 방식: `(col == 1).sum()` 으로 컬럼별 응답 수 카운트.
컬럼 순서와 항목 매핑:
- `경영_애로사항1코드` = 상권 쇠퇴
- `경영_애로사항2코드` = 동일 업종 경쟁심화
- `경영_애로사항3코드` = 원재료비·재료매입비
- `경영_애로사항4코드` = 최저임금 영향(인건비)
- `경영_애로사항5코드` = 보증금·월세
- `경영_애로사항6코드` = 부채상환
- `경영_애로사항7코드` = 인력관리(채용의 어려움 등)
- `경영_애로사항8코드` = 판로 개척 및 운영(온라인 등)
- `경영_애로사항9코드` = 디지털 기술·기기 도입 및 운영
- `경영_애로사항10코드` = 기타 (집계 제외)

### 디지털 대응 코드 (경영_운영활동_디지털대응1~8코드)
**컬럼 구조**: 각 컬럼이 1/NaN 플래그.

## 대시보드 구현 현황

**기술 스택**: Python (pandas) → JSON 인라인 삽입 → 단일 `index.html`
- 차트 라이브러리: Chart.js 4.4.1 (CDN)
- `analysis.py` 실행 시 `index.html`의 `const D = {...};` 라인만 교체 (UI 코드 보존)

### 현재 구조: 7섹션 탐색형 대시보드

#### 분석 필터
- **대상**: 음식점업 (`산업대분류코드=='I'` AND `산업중분류코드==56`)
- **표본**: 1,162개 (매출>0 기준)

#### 섹션 구성

**① 산업 개요**
- KPI 카드: 표본 수·추정사업체수·영업이익률·월매출·월영업이익·폐업의향
- 차트: 종사자규모 분포 / 매출구간 분포 / 운영계획 도넛 / 애로사항 9항목 가로막대

**② 비용구조**
- 지표 토글 (영업이익률 / 월 영업이익 / 월 매출 / 폐업 의향)
- 차트: 종사자규모별 누적 비용구조 / 매출구간별 누적 비용구조
- 교차표: 매출구간 × 종사자규모 (선택 지표값 표시)

**③ 종사자규모 × 고용형태**
- 지표 토글
- 차트: 고용형태 구성 비율 / 규모×고용형태별 지표 grouped bar
- 교차표: 종사자규모 × 고용형태 (선택 지표값 표시)
- 버블차트: x=영업이익률, y=월 영업이익, 버블크기=폐업의향

**④ 프랜차이즈 × 고용형태**
- 차트: 영업이익률 비교 / 폐업의향 비교
- 교차표: 프랜차이즈 × 고용형태
- 레이더차트: 4개 축(영업이익률·월이익·월매출·폐업안정성), 정규화 0~100, n≥10 셀만

**⑤ 대표자 연령대**
- 지표 토글
- 차트: 연령대별 선택지표 / 폐업의향 / 주요 애로사항 라인(3종) / 디지털·배달 현황
- 연령대 코드: 30=30대, 40=40대, 50=50대, 60=60대, 70=70대+

**⑥ 창업경험 × 운영연수**
- 지표 토글
- 차트: 운영연수별 지표 / 창업횟수별 지표
- 교차표 2종: 창업횟수 × 운영연수 (min_n=5) / 운영연수 × 고용형태 (min_n=10)

**⑦ 지역분석**
- 지표 토글
- 차트: 지역별(수도권·광역시·기타) 선택지표 / 폐업의향 / 표본 분포 도넛 / 주요 애로사항
- 버블차트: 시도별(~15개, n≥20) — x=임차료율, y=월 영업이익, 버블크기=추정사업체수
  - 색상: 수도권(#1e261b) / 광역시(#587638) / 기타(#95a961)
  - 버블 위 시도명 레이블: `afterDraw` 인라인 플러그인으로 렌더

#### 지표 토글 시스템
- 4개 지표: `pr`(영업이익률) / `mp`(월 영업이익) / `ms`(월 매출) / `cl`(폐업 의향)
- 섹션별 독립 상태 (`secMetric` 객체)
- 토글 전환 시 해당 섹션 차트 destroy → 재생성, 교차표 재빌드
- `.mtbtn.active`: `--green-2` 배경 + `--green-5` 텍스트

#### 차트 레지스트리
- `charts` 객체에 canvas id별 Chart 인스턴스 저장
- `newChart(id, config)`: 기존 인스턴스 destroy 후 재생성

#### 교차표 `buildXtab` n 부족 처리
- `cell.n === 0` → `-` 표시 (회색)
- `cell.n > 0` but 지표값 `null` (min_n 미달) → `n 부족` + `n=X` 표시 (opacity 0.45)
- 정상 셀 → 지표값 + `폐업 X%` 부제목

#### 애로사항 데이터 구조 (`hardship`)
- `"기타"` 항목 제외, 9개 항목만 사용
- `{"labels": [...], "rates": [%값], "counts": [건수]}`
- 툴팁: `XX% (N건 / n명)` 형태로 원본 건수 병기

## 세그먼트 경계값 처리 기준

매출 구간은 **초과~이하** 기준으로 분류한다 (단위: 백만원).

| 구간 | 조건 |
|------|------|
| L1 | 0 < 매출 ≤ 36 |
| L2 | 36 < 매출 ≤ 72 |
| L3 | 72 < 매출 ≤ 120 |
| L4 | 매출 > 120 |

경계값(36, 72, 120)에 해당하는 케이스는 **하위 구간에 포함**된다.
`analysis.py` 수정 시 이 기준을 반드시 유지할 것.

## analysis.py 구현 패턴

### 파생 컬럼 벡터화 방식
모든 파생 세그먼트 컬럼은 `apply()` 대신 벡터화 연산으로 처리한다.
- `pd.cut()`: 구간 분류 (종사자규모·매출구간·운영연수구간·창업횟수구간)
- `np.select()`: 조건 분기 (고용형태·지역)
- `.map()`: 코드→레이블 1:1 매핑 (연령대·프랜차이즈)

### HCOLS 사전 필터링
`HCOL_MAP`에서 실제 존재하는 컬럼만 모듈 레벨에서 `HCOLS`로 1회 필터링.
`cnt_hard(sub)` 내부에서 매번 `col in sub.columns` 체크하지 않음.

### SEC7 시도별 집계
```python
CITY_MAP = {11:'서울', 21:'부산', ..., 39:'제주'}   # 17개 시도
code_col = pos["행정구역시도코드"].fillna(-1).astype(int)
# n≥20 통과 시 stats["region_type"], stats["is_metro"] 추가
```
`sec7["by_city"]`, `sec7["city_order"]` 로 접근.

## Known Issues

- **연령대 코드값**: `대표자연령대코드`의 실제 저장값은 30·40·50·60·70 (정수). 1~6 같은 순서 코드가 아님. `AGE_MAP = {30:'30대', ...}` 로 매핑해야 한다.
- **디지털 미도입 판별**: `경영_운영활동_디지털대응8코드 == 1`이 "활동사항 없음" 플래그. 코드값 54가 직접 저장되는 구조가 아님 — 8번 컬럼의 1/NaN 플래그 구조.
- **4~9인 표본 소규모**: `4~9인` 그룹 n=63으로 통계적으로 불안정. 참고용으로만 활용.
- **인건비율 0% 정상값**: 소규모 가게의 다수가 직원 없이 대표자만 운영 → 전체 중앙값 0%는 정상. 버그가 아님.

## 주의사항
- 매출액이 0이거나 이상값(매출액 > 100억 = 10,000백만원)은 분석 전 제거
- 비율 계산 시 매출액이 0인 행 제외
- 가중치(`사업체수가중값`) 적용 여부를 항상 명시할 것
- 금액 단위: CSV는 백만원 / 대시보드 표시는 만원(×100), 월 기준은 ÷12 추가 적용
- 코드값은 반드시 한글 레이블로 변환하여 출력
- `analysis.py`는 `index.html`의 `const D = {...};` 라인만 교체 — HTML/CSS/JS 코드 보존
