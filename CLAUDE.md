# 소상공인 실태조사 데이터 분석 프로젝트

## 프로젝트 배경
대한민국 자영업자(특히 영세 요식업)를 장기적으로 돕는 사업의 기반을 마련하기 위한 탐색 프로젝트.
이 분석의 목적은 "자영업자가 실제로 겪는 문제 중 내가 풀 수 있는 의미 있는 문제는 무엇인가"를 데이터로 탐색하는 것이다.

## 분석 목표 (우선순위 순)

### 1. 타겟 세그먼트 규모 파악
- 숙박·음식점업 (`산업대분류코드 == 'I'`) 필터링
- 종사자 수 구간별 분포: 1명(대표자만), 2~3명, 4~9명, 10명 이상
- **핵심 타겟**: 종사자 1~3명 구간의 절대 수 및 비율

### 2. 비용구조 분석 (종사자 규모별 비교)
- 매출 대비 각 비용 항목 비율 계산 (중앙값 기준)
  - 원가율 = 매출원가 / 매출액
  - 인건비율 = 급여총액 / 매출액
  - 임차료율 = 임차료 / 매출액
  - 기타비용율 = 기타금액 / 매출액
  - 영업이익률 = 영업이익 / 매출액
- 종사자 1~3명 vs 4명 이상 비교
- 매출 구간별(~6,400만 / 6,400만~1억 / 1억~2억 / 2억 이상) 비용구조 분포
  - 기준값 (백만원): 64 / 100 / 200
- **인건비율 주의**: 직원 없는 가게(소규모의 60.5%)는 인건비율=0 → 전체 중앙값은 0%.
  직원 있는 가게(39.5%) 내 인건비율 중앙값은 22.2%로 별도 표시

### 3. "잘 버티는 가게" vs "못 버티는 가게" 비교
- 영업이익률 상위 25% vs 하위 25% 집단 비교 (Q25=14.3%, Q75=41.8%)
- 어떤 비용 항목에서 차이가 나는가
- 디지털 도입 여부, 프랜차이즈 여부, 임차 여부 등 운영 특성 차이

### 4. 애로사항 분석
- 숙박·음식점업 전체에서 애로사항 빈도 순위
- 종사자 규모별 애로사항 차이
- 영업이익률 하위 집단의 주요 애로사항

### 5. 폐업 의향 분석
- 사업전환_운영계획코드 별 분포 (가중치 적용)
- 폐업 의향 집단의 특성 (매출, 이익률, 애로사항)

## 대시보드 구현 현황

**기술 스택**: Python (pandas) → JSON 인라인 삽입 → 단일 `index.html`
- 차트 라이브러리: Chart.js 4.4.0 (CDN)
- `analysis.py` 실행 시 `index.html`의 `const D = {...};` 라인만 교체 (UI 코드 보존)
  - HTML 템플릿을 재생성하지 않음 — index.html을 직접 수정한 내용이 유지됨

### 현재 구조: 16세그먼트 히트맵 대시보드

#### 세그먼트 정의
- **종사자 축 (S1~S4)**: S1=1명 / S2=2~3명 / S3=4~9명 / S4=10명+
- **매출 축 (L1~L4)**: L1=~3,600만 / L2=3,600만~7,200만 / L3=7,200만~1.2억 / L4=1.2억+
- S3·S4는 핵심 타겟 외 참고용 (opacity 0.35)

#### 히트맵 셀 정보
- 영업이익률 중앙값 → 5단계 그린 색상 (< 10% / 10~20% / 20~30% / 30~40% / 40%+)
- 표본 수 (n) 표시, 최소 5개 이상인 셀만 활성화

#### 상세 패널 (하단)
셀 선택 시 표시되는 정보:
- 표본 수 / 추정 사업체 수 / 영업이익률 / 매출 중앙값 / 폐업 의향
- 비용 구조 가로 막대 차트 (원가율·인건비율·임차료율·기타비용율·영업이익률)
- 경영 애로사항 9개 항목 가로 막대 차트

#### 선택 모드 (LEFT / RIGHT)
- 히트맵 위 모드 토글 버튼으로 현재 활성 패널 전환
  - LEFT: `--green-2(#3a5630)`, RIGHT: `--amber(#b45309)`
- 셀 클릭 시 현재 활성 모드 패널에 담김 (toggle). 반대편에 이미 있으면 이동
- 셀 배지: LEFT 선택 → 좌상단 `L` 뱃지 + 초록 테두리, RIGHT 선택 → 우상단 `R` 뱃지 + 주황 테두리
- **행·열 헤더 클릭**: 현재 활성 모드로 해당 행/열 전체 유효 셀 일괄 담기 (이미 전부 선택 시 해제)
- **헤더 시각 피드백**: hover + L/R 색상 outline

#### 하단 2패널 비교 구조
- LEFT / RIGHT 패널 나란히 배치
- 선택된 셀 없는 패널은 "셀을 선택해주세요" 안내
- **집계 방식**:
  - 표본 수 / 추정 사업체 수: 합산
  - 영업이익률 / 각 비용율 / 폐업 의향: 표본 수(n) 기준 가중평균
  - 애로사항: counts 합산 후 결합 n 기준 비율(%) 재계산, 전체 9개 항목 표시
- **delta 표시** (RIGHT 패널에만): `▲▼Δ%p` (RIGHT − LEFT 차이, 양수=초록·음수=빨강)
- 각 패널: 스탯 테이블 + 비용 구조 차트 + 애로사항 9개 항목 차트
- **기본 선택**: LEFT = S1×L2, RIGHT = S2×L2

#### 차트 축 동기화
- **비용 구조 차트**: x축 0~100% 고정
- **애로사항 차트**: `Math.max(leftMax, rightMax)` 기준 양쪽 x축 동기화 (%, 선택 변경 시 재계산)

#### 애로사항 데이터 구조 (`hardship`)
- `"기타"` 항목 제외, 9개 항목만 사용
- Python에서 비율로 저장: `{"labels": [...], "rates": [%값], "counts": [건수]}`
- JS 집계: 복수 셀 선택 시 `counts` 합산 후 결합 n으로 비율 재계산
- 툴팁: `XX% (N건 / n명)` 형태로 원본 건수 병기
- 차트 높이: 280px (9개 항목 수용)

### Chart.js 전역 설정
```javascript
Chart.defaults.font.family = "'Apple SD Gothic Neo', 'Malgun Gothic', 'Noto Sans KR', sans-serif";
Chart.defaults.color = '#6b6b62';
// 그리드: y축 #f2f2ed, x축 display:false
```

### 디자인 원칙
1. **그린 중심**: 모든 강조·액션은 브랜드 그린 팔레트 안에서 해결. 파란색 계열 강조 지양.
2. **배경 계층 3단계**: 페이지(`--neutral-100`) → 컨텐츠(`--neutral-0`) → 인라인 강조(`--neutral-100` 또는 `--green-5`)
3. **텍스트 위계**: `--neutral-900` → `700` → `500` → `300`
4. **그림자 최소화**: 카드 구분은 테두리·배경색 차이로. 플로팅 요소에만 shadow 사용.
5. **의미 기반 색상**: 색상은 순서가 아니라 의미로 할당. 같은 항목은 항상 같은 색.
6. **모바일 우선**: 720px 이하 단일 컬럼 전환.

## 주의사항
- 매출액이 0이거나 이상값(매출액 > 100억 = 10,000백만원)은 분석 전 제거
- 비율 계산 시 매출액이 0인 행 제외
- 가중치(`사업체수가중값`) 적용 여부를 항상 명시할 것
- 금액 단위는 백만원 기준으로 통일, 차트에 단위 명기
- 코드값은 반드시 한글 레이블로 변환하여 출력
- 디지털 대응 코드 54(활동없음)는 `NO_DIG` 집합으로 처리 (코드 8, 54 모두 제외)
