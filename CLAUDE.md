# 소상공인 실태조사 데이터 분석 프로젝트

## 프로젝트 배경
대한민국 자영업자(특히 영세 요식업)를 장기적으로 돕는 사업의 기반을 마련하기 위한 탐색 프로젝트.
이 분석의 목적은 "자영업자가 실제로 겪는 문제 중 내가 풀 수 있는 의미 있는 문제는 무엇인가"를 데이터로 탐색하는 것이다.

### 애로사항 코드 (경영_애로사항1~10코드)
**컬럼 구조**: 각 컬럼이 항목별 1/NaN 플래그 (1=해당, NaN=미해당) — 코드값이 아님.
집계 방식: `(col == 1).sum()` 으로 컬럼별 응답 수 카운트.
컬럼 순서와 항목 매핑:
- `경영_애로사항1코드` = 상권 쇠퇴
- `경영_애로사항2코드` = 동일 업종 경쟁심화
- `경영_애로사항3코드` = 원재료비·재료매입비
- `경영_애로사항4코드` = 최저임금 영향(인건비)
- `경영_애로사항5코드` = 보증금·월세
- `경영_애로사항6코드` = 부채상환
- `경영_애로사항7코드` = 인력관리(채용의 어려움 등)
- `경영_애로사항8코드` = 판로 개척 및 운영(온라인 등)
- `경영_애로사항9코드` = 디지털 기술·기기 도입 및 운영
- `경영_애로사항10코드` = 기타

## 분석 목표 (우선순위 순)

### 1. 타겟 세그먼트 규모 파악
- 숙박·음식점업 (`산업대분류코드 == 'I'`) 필터링
- 종사자 수 구간별 분포: 1명(대표자만), 2~3명, 4~9명, 10명 이상
- **핵심 타겟**: 종사자 1~3명 구간의 절대 수 및 비율

### 2. 비용구조 분석 (종사자 규모별 비교)
- 매출 대비 각 비용 항목 비율 계산 (중앙값 기준)
  - 원가율 = 매출원가 / 매출액
  - 인건비율 = 급여총액 / 매출액
  - 임차료율 = 임차료 / 매출액
  - 기타비용율 = 기타금액 / 매출액
  - 영업이익률 = 영업이익 / 매출액
- 종사자 1~3명 vs 4명 이상 비교
- 매출 구간별(~6,400만 / 6,400만~1억 / 1억~2억 / 2억 이상) 비용구조 분포
  - 기준값 (백만원): 64 / 100 / 200
- **인건비율 주의**: 직원 없는 가게(소규모의 60.5%)는 인건비율=0 → 전체 중앙값은 0%.
  직원 있는 가게(39.5%) 내 인건비율 중앙값은 22.2%로 별도 표시

### 3. "잘 버티는 가게" vs "못 버티는 가게" 비교
- 영업이익률 상위 25% vs 하위 25% 집단 비교 (Q25=14.3%, Q75=41.8%)
- 어떤 비용 항목에서 차이가 나는가
- 디지털 도입 여부, 프랜차이즈 여부, 임차 여부 등 운영 특성 차이

### 4. 애로사항 분석
- 숙박·음식점업 전체에서 애로사항 빈도 순위
- 종사자 규모별 애로사항 차이
- 영업이익률 하위 집단의 주요 애로사항

### 5. 폐업 의향 분석
- 사업전환_운영계획코드 별 분포 (가중치 적용)
- 폐업 의향 집단의 특성 (매출, 이익률, 애로사항)

## 대시보드 구현 현황

**기술 스택**: Python (pandas) → JSON 인라인 삽입 → 단일 `index.html`
- 차트 라이브러리: Chart.js 4.4.0 (CDN)
- `analysis.py` 실행 시 `index.html` 자동 생성 (외부 파일 의존 없음)

**구성 섹션**:
1. **① 세그먼트 규모**: 요약 카드 5개 + 종사자 분포 막대/도넛 차트
2. **② 비용구조**: 종사자 규모별 / 소규모vs대규모 / 매출 구간별 토글 전환
3. **③ 수익성 분포**: 영업이익률 히스토그램 (전체·상위25%·하위25%) + 비교표
4. **④ 애로사항 Top 5**: 소규모 / 대규모 / 전체 / 이익률 하위25% 토글 전환
5. **⑤ 사업 전망**: 운영계획 도넛 + 폐업의향 집단 애로사항 + 계속운영 vs 폐업의향 비교표

## 주의사항
- 매출액이 0이거나 이상값(매출액 > 100억 = 10,000백만원)은 분석 전 제거
- 비율 계산 시 매출액이 0인 행 제외
- 가중치(`사업체수가중값`) 적용 여부를 항상 명시할 것
- 금액 단위는 백만원 기준으로 통일, 차트에 단위 명기
- 코드값은 반드시 한글 레이블로 변환하여 출력
- 디지털 대응 코드 54(활동없음)는 `NO_DIG` 집합으로 처리 (코드 8, 54 모두 제외)
